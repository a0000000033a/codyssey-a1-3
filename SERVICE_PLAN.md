# 밥밥 서비스 기획서

## 서비스 목적

밥밥은 사용자가 오늘의 상황을 문장으로 입력하면 AI가 어울리는 메뉴를 추천하고, 추천 메뉴를 기준으로 가까운 음식점을 찾아주는 서비스입니다. 메뉴를 고르는 시간을 줄이고, 바로 실행 가능한 식사 선택을 돕는 것이 목적입니다.

## 타겟 사용자

- 점심이나 저녁 메뉴를 빠르게 정하고 싶은 사람
- 기분, 날씨, 예산에 맞는 메뉴 추천을 받고 싶은 사람
- 추천받은 메뉴를 근처에서 바로 먹고 싶은 사람

## 페이지 구성

1. AI 프롬프트 입력 페이지: 사용자가 자유 텍스트로 상황을 입력합니다.
2. AI 답변 페이지: 추천 메뉴, 추천 이유, 검색 키워드, 태그를 보여줍니다.
3. 카카오 맵 추천 페이지: 추천 메뉴 키워드로 주변 음식점 최대 5개를 보여줍니다.

## 핵심 기능

- 사용자 프롬프트 입력
- 프롬프트 기본 정제 및 길이 제한
- OpenAI API를 통한 메뉴 추천
- JSON 응답 파싱 및 정규화
- 브라우저 Geolocation 기반 위치 확인
- 위치 권한 거부 시 개포동역 기본 위치 사용
- Kakao Local API 음식점 검색

## AI 기능 기준

### 입력

사용자는 300자 이내의 자유 텍스트를 입력합니다. 예시는 다음과 같습니다.

```text
비 오는 날 따뜻하고 든든한 점심을 먹고 싶어. 너무 비싸지 않았으면 좋겠어.
```

### 출력

백엔드는 AI에게 아래 JSON 형식을 요청하고, 응답을 다시 검증합니다.

```json
{
  "menu": "김치찌개",
  "reason": "따뜻하고 든든하며 비 오는 날에 잘 어울리는 한식 메뉴입니다.",
  "searchKeyword": "김치찌개",
  "tags": ["따뜻함", "든든함", "한식"]
}
```

`menu`와 `reason`은 필수입니다. `searchKeyword`는 Kakao 음식점 검색에 사용합니다.

### 실패 처리

- 빈 입력: 사용자에게 입력 필요 메시지를 보여줍니다.
- AI API 오류: 추천 처리 실패 메시지를 보여줍니다.
- AI JSON 파싱 실패: 응답 형식 오류 메시지를 보여줍니다.
- 응답 지연: 20초 후 지연 안내 메시지를 보여줍니다.
- 위치 권한 거부: 개포동역 기준으로 검색합니다.
- 음식점 결과 없음: 조건에 맞는 음식점이 없다고 안내합니다.

## 어뷰징 방지 기준

- 프론트와 백엔드에서 입력 길이를 제한합니다.
- `<`, `>` 문자를 제거해 화면 출력과 요청에 들어가는 위험 문자를 줄입니다.
- 시스템 프롬프트에서 역할 변경, 키 요청, 정책 우회 요구를 무시하도록 지시합니다.
- AI 응답은 JSON으로만 받도록 요청합니다.
- 백엔드에서 필수 필드와 타입을 검증한 뒤 프론트로 전달합니다.
